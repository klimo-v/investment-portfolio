# Investment Portfolio — учёт сделок и портфелей

Личное приложение для учёта инвестиционных сделок по нескольким торговым системам
и брокерам. Заменяет ручное ведение в Google Sheets. На старте — локальный веб на
своём ПК, с заложенной возможностью расширения до десктоп-приложения и публичного
SaaS без переписывания кода.

## Проблема, которую решаем

Сейчас сделки из разных брокеров вносятся вручную в три листа Google Sheets
(Сделки, Закрытые сделки, Портфель), причём два из них дублируются руками. Это
долго и приводит к ошибкам. Цель — вводить/импортировать данные **в одном месте**,
а позиции, сделки и итоги считать **автоматически**.

## Стек (кратко)

TypeScript везде. Фронтенд — **Angular 22** (standalone, signal-first, zoneless) +
Angular Material + Signal Forms. Бэкенд — **NestJS** (Fastify-адаптер) + Drizzle ORM
+ SQLite (→ PostgreSQL в SaaS). Валидация — Zod. Монорепо — Nx. Десктоп позже —
Tauri 2. Подробности в `docs/01-tech-stack.md`.

## Ключевые решения

- **Один источник истины** — журнал операций; позиции и сделки вычисляются, не
  дублируются руками.
- **Деньги — не во float** — decimal/целые единицы, чтобы не терять копейки.
- **Импорт как плагины** — по одному парсеру на брокера, общее ядро.
- **Рыночные данные по API** — MOEX ISS, ЦБ РФ, крипто-API.
- **Тесты движка расчётов на реальных цифрах** — защита от ошибок в формулах.

## Глоссарий

- **Система** — торговая стратегия (Верников, Верников_трейдинг, Трейдинг_крипта).
- **Портфель** — брокерский счёт / назначение (позволяет вести переводы между ними).
- **Операция** — одна строка журнала (покупка, продажа, дивиденд, налог, перевод…).
- **Сделка (Trade)** — группа операций по инструменту; бывает открытой/закрытой.
- **Позиция** — текущий остаток по инструменту в портфеле.
