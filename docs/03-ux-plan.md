# 03 — UX / юзабилити: как переоформить страницы

Пошаговый разбор ваших текущих трёх листов: что в них неудобно и как это должно
выглядеть в приложении. Цель — **ввод в одном месте, всё остальное считается само**,
и удобная навигация вместо горизонтальной прокрутки на 18 колонок.

---

## Общий принцип

Сейчас вы работаете в «плоских» таблицах, где всё в одном ряду и всё вводится
руками. В приложении разделяем **ввод** и **просмотр**:

- **Ввод** — короткие умные формы и импорт файлов.
- **Просмотр** — дашборд и таблицы с фильтрами, сортировкой, группировкой,
  где ничего не надо считать самому.

Структура приложения (боковое меню):

```
📊 Дашборд          — сводка по всем системам, ключевые цифры, графики
💼 Портфель         — текущие позиции (= лист «Портфель ОБЩИЙ»)
📁 Сделки           — закрытые/открытые сделки (= лист «Закрытые сделки»)
📝 Операции         — журнал (= лист «Сделки»), сюда вводим/импортируем
📥 Импорт           — загрузка отчётов брокеров
⚙️  Справочники      — системы, портфели, инструменты
```

## Шаг 1. Лист «Сделки» → страница «Операции» + форма ввода

### Что неудобно сейчас

- 18 колонок в одну строку — горизонтальная прокрутка, легко ошибиться колонкой.
- Часть колонок вычисляемые (Итого, Итого RUB), но вводятся/поддерживаются руками.
- Тип операции (Deposit/Buy/Sell/Tax…) в одной колонке с разной логикой полей.
- Легко забыть комиссию, курс, привязку Trade ID.

### Как сделать

**Ввод — через модальную форму «Добавить операцию», а не прямо в таблицу.**
Форма **адаптивная**: показывает только нужные поля под выбранный тип операции.

- Выбрали **Buy/Sell** → поля: инструмент, количество, цена, комиссия, дата,
  система, портфель. Сумма и «Итого» считаются на лету и показываются, но **не
  вводятся**. Курс к RUB подтягивается автоматически на дату, с возможностью
  переопределить.
- Выбрали **Deposit/Withdraw** → только сумма, валюта, портфель, дата.
- Выбрали **Dividend/Coupon/Tax** → инструмент (для дивиденда/купона), сумма, дата.
- Выбрали **Transfer (между портфелями)** → «откуда», «куда», сумма — создаётся
  сразу пара связанных операций.

**Просмотр — таблица операций** с:

- фильтрами по системе, портфелю, инструменту, типу, диапазону дат;
- сортировкой по любой колонке;
- поиском по тикеру;
- цветовыми метками типов (зелёный Buy, красный Sell, синий Dividend…);
- скрытием технических колонок по умолчанию (разворачиваются по клику на строку).

**Что улучшается:** вместо 18 колонок сразу видно 6–7 главных; вычисляемые поля не
вводятся руками; ошибиться колонкой невозможно.

## Шаг 2. Лист «Закрытые сделки» → страница «Сделки» (только просмотр)

### Что неудобно сейчас

- Поддерживается вручную параллельно с листом «Сделки» — двойной ввод, риск
  рассинхрона.
- P&L и статус считаются формулами, которые надо тянуть на каждую новую строку.

### Как сделать

**Эта страница ничего не вводит** — она полностью строится из журнала операций.

- Карточки/строки сделок со **статусом** (Открыта / Частично / Закрыта) — цветной бейдж.
- Две вкладки: **Открытые** и **Закрытые**.
- На строке видно: тикер, система, вложено, дивиденды/купоны, текущая стоимость,
  P&L (₽ и %). P&L красный/зелёный.
- Клик по сделке → раскрывается список **операций**, из которых она собрана
  (какие покупки/продажи/выплаты вошли). Полная прослеживаемость.
- Итоговая строка снизу: суммарный P&L по фильтру.

**Что улучшается:** исчезает двойной ввод; сделка всегда согласована с операциями;
видно, из чего именно она собрана.

## Шаг 3. Лист «Портфель ОБЩИЙ» → страница «Портфель»

### Что неудобно сейчас

- 16 колонок, часть отрицательных P&L теряется в общем полотне.
- Один общий лист + копии по портфелям = ручное дублирование одной и той же логики.
- Не видно структуры портфеля (доли, разбивка по типам/валютам) без отдельных
  сводных таблиц.

### Как сделать

Одна страница «Портфель» с **переключателем контекста сверху**:

```
[ Все системы ▾ ]  [ Все портфели ▾ ]  [ Валюта: RUB ▾ ]
```

Меняете фильтр — та же таблица показывает общий портфель, или только «Верников»,
или только Tinkoff. Отдельные листы-копии больше не нужны.

Сверху — **плашки-итоги**: Вложено, Текущая стоимость, P&L ₽, P&L %, Дивиденды+купоны.

Таблица позиций:

- группировка по **типу инструмента** (Stock / Bond / ETF / Crypto / Currency) со
  свёрткой групп;
- колонки: тикер, кол-во, средняя цена, текущая цена, вложено, тек. стоимость,
  P&L ₽, P&L %, доля в портфеле;
- P&L цветом; доля — маленькой полоской-баром прямо в ячейке;
- сортировка по любой колонке (например, «худшие позиции сверху»).

Справа/сверху — **круговая диаграмма** структуры (по типам или по тикерам).

**Что улучшается:** один экран вместо множества листов-копий; сразу видно структуру
и проблемные позиции; переключение системы/портфеля в один клик.

## Шаг 4. Новое: страница «Дашборд»

Чего сейчас нет вообще, но что даст максимум пользы — **стартовый экран-сводка**:

- Ключевые цифры: суммарная стоимость всех портфелей, общий P&L, доход за месяц.
- **P&L по системам** (столбики) — сравнить «Верников» vs «Трейдинг_крипта».
- **Динамика стоимости портфеля** во времени (линия) — если хранить снимки.
- Топ-5 прибыльных / топ-5 убыточных позиций.
- Ближайшие события: выплаты купонов/дивидендов (для облигаций).

---

## Шаг 5. Новое: страница «Импорт»

Реализует ваш сценарий «выгрузка из брокера → файл → программа сама разбирает»:

1. Перетащить файл отчёта (.xlsx/.csv), выбрать брокера и портфель.
2. Программа парсит и показывает **предпросмотр**: сколько операций распознано, как
   классифицированы (покупка / перевод между портфелями / дивиденд…).
3. Строки с сомнительной классификацией подсвечиваются — можно поправить вручную.
4. Дубли с уже загруженными автоматически отсекаются и помечаются.
5. Кнопка «Импортировать» → операции попадают в журнал, портфель пересчитывается.
6. Любую загрузку можно **откатить** целиком (по `import_batch_id`).

## Сквозные улучшения юзабилити

- **Числа:** разделители разрядов, выравнивание по правому краю, 2 знака для
  валют; отрицательные — красным. Никаких «497,782.00000000000».
- **Валюта:** значок валюты у суммы; переключатель отображения RUB / валюта сделки.
- **Цвет:** прибыль зелёным, убыток красным — единообразно везде.
- **Ввод дат:** календарь-пикер, единый формат, без ручного `2/27/2025`.
- **Автоподстановка:** тикеры и инструменты выбираются из справочника (автокомплит),
  а не печатаются заново — исключает опечатки вроде разных написаний одного тикера.
- **Горячие клавиши:** быстрый «Добавить операцию».
- **Экспорт:** выгрузка любой таблицы обратно в Excel/CSV (обратная совместимость).
- **Undo:** отмена последнего действия.

---

## Приоритет для MVP

Обязательно в MVP: **Операции (ввод + таблица)**, **Портфель**, **Сделки**, базовый
**Импорт**. Дашборд и графики — приятны, но их можно упростить в первой версии и
развить в v1 (см. `04-roadmap.md`).
