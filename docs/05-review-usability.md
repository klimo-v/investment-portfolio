# 05 — Независимое ревью: практичность и удобство

Дата: 2026-07-24. Ревью составлено по фактическому коду (`apps/web`, `apps/api`,
`libs/core`), а не по планам из roadmap. Оценка — с позиции реального пользователя,
который ведёт несколько торговых систем и портфелей и хочет **понимать
эффективность**, а не просто хранить журнал.

Дисклеймер: это продуктово-UX ревью приложения. Это не персональная инвестиционная
рекомендация.

Итоговое впечатление: **крепкий, аккуратный MVP**. Журнал → позиции → сделки →
дашборд собраны честно (единый источник истины, движок в `@core`, деньги в decimal).
Но приложение сегодня отвечает на вопрос «**сколько у меня денег и P&L в рублях**» и
почти не отвечает на вопрос «**насколько эффективна каждая система/портфель и стоит
ли её продолжать**». Именно этот разрыв — главный резерв.

---

## 1. Фичи для оценки эффективности системы/портфеля — да, нужны (ключевой пробел)

Сейчас везде считается P&L **только в абсолютных рублях** (`totals.pnlRub`,
`bySystem.pnlRub`, breakdown по тикерам). Абсолютный P&L не позволяет сравнивать
системы между собой: система с +100 000 ₽ на вложенные 3 млн хуже, чем +60 000 ₽ на
500 тыс. Для «оценки эффективности» не хватает относительных и риск-метрик.

Чего нет и что критично добавить (по убыванию важности):

1. **Доходность в % и годовых (XIRR / TWR).** В `summary.bySystem` уже есть и
   `investedRub`, и `pnlRub` — простейший ROI = `pnlRub / investedRub` можно показать
   сразу, почти бесплатно. Но корректная метрика при вводах/выводах — **XIRR**
   (денежно-взвешенная) или **TWR** (для сравнения самой стратегии без влияния
   таймингов пополнений). Данные для XIRR уже есть: даты и суммы всех Deposit/
   Withdraw/Dividend + текущая стоимость на «сегодня». Это самая полезная фича из
   всех перечисленных.
2. **Сравнение с бенчмарком.** «Эффективна ли система» — вопрос относительно
   альтернативы: индекс MOEX (IMOEX уже доступен через тот же MOEX ISS коннектор),
   ставка депозита, инфляция. Линия «портфель vs IMOEX» на графике динамики
   (снимки уже есть) отвечает на него наглядно.
3. **Статистика по сделкам для торговых систем.** Страница «Сделки» показывает
   список, но **не агрегаты**. Для оценки стратегии нужны: win rate (% прибыльных),
   profit factor (сумма прибылей / сумма убытков), средняя прибыль/убыток, матожидание
   на сделку, средний срок удержания, число сделок. Движок `calculateTrades` уже даёт
   `realizedPnlRub`, `openedAt/closedAt` — всё для расчёта есть, нужен слой агрегации
   **в разрезе системы**.
4. **Реализованный vs нереализованный P&L раздельно.** Движок считает
   `realizedPnlCcy`, но в UI (дашборд, портфель) показан только суммарный P&L. Для
   оценки системы это разные вещи: реализованный — что стратегия уже заработала,
   нереализованный — бумажная переоценка открытых позиций.
5. **Риск-метрики из снимков.** Снимки стоимости уже пишутся — из них почти даром
   считается **максимальная просадка (max drawdown)** и волатильность equity-кривой.
   Для «эффективности» доходность без риска — половина картины.
6. **Дивидендная доходность в %** (полученные выплаты / вложено), а не только сумма ₽.

Мини-приоритет: пункты 1 и 3 дают максимум смысла при минимуме кода (данные уже
посчитаны движком) — начать с них.

## 2. Оформление интерфейса — чисто, но функционально «тонко»

Плюсы: последовательный Angular Material, предсказуемая навигация (sidenav + 5
разделов), везде есть пустые состояния и состояния загрузки/ошибки, подсказка «?» с
пояснением про дивиденды на «Портфеле» — хороший, заботливый штрих. Визуально
опрятно.

Слабые места (реально мешают в работе):

- **Фильтры только на «Портфеле».** На Дашборде, Операциях и Сделках нельзя
  отфильтровать по системе/портфелю/периоду. Дашборд показывает «всё вместе», а
  пользователь ведёт **несколько** систем — это первый же запрос «покажи только
  систему X». Нет и выбора периода (YTD / 1 год / всё).
- **Таблица «Операции» не масштабируется.** Нет сортировки, поиска, пагинации,
  фильтра по дате/типу/тикеру и виртуального скролла. На сотнях-тысячах строк журнала
  это станет больно — при том что CLAUDE.md §9 сам предписывает CDK Virtual Scroll.
- **Нельзя отредактировать операцию.** Доступны только удаление и переназначение
  системы/портфеля. Опечатку в цене/количестве приходится удалять и вводить заново.
- **Таблица «Сделки» без сортировки/фильтра** и без строки-итога с агрегатами
  (см. §1.3).
- **Нет тёмной темы, цвета захардкожены.** `background: #fafafa`, `th { background:
  white }`, `rgba(0,0,0,0.6)` в стилях — при тёмной теме текст на белом «поедет».
  CLAUDE.md заявляет темы (`libs/web-shared`), но по факту это светлая тема с
  инлайновыми цветами вместо токенов.
- **Отображение нуля как «—».** В `portfolio.page.ts` условия вида
  `p.pnlRub ? … : '—'` показывают прочерк и при честном нуле P&L (0 — falsy), а не
  только при отсутствии данных. Мелочь, но вводит в заблуждение.
- **Тексты ошибок «девелоперские».** «Не удалось загрузить… Запущен ли API?» — это
  сообщение для разработчика, не для пользователя.
- **Адаптивность под мобильный** не проверена: `.totals` (flex-ряд карточек) и
  `.content` (grid 2fr/1fr) на узком экране, скорее всего, ломаются.
- **Нет подтверждения на удаление** выбранных операций (стоит проверить
  `removeSelected` — массовое удаление без «вы уверены?» опасно).

## 3. Специальные фичи, которые стоит внедрить

Отсортировано по соотношению «польза / усилия»:

**Быстро и очень полезно (данные уже есть в движке/summary):**
- **Карточки/таблица эффективности по системам и портфелям**: вложено, стоимость,
  P&L ₽ **и %**, доля от общего портфеля. Основа — существующий `bySystem`.
- **Блок статистики сделок** на странице «Сделки»: win rate, profit factor, средний
  выигрыш/проигрыш, средний срок удержания, число закрытых сделок — с фильтром по
  системе.
- **Глобальный фильтр «система / портфель / период»** в шапке дашборда.
- **Экспорт таблиц в CSV/Excel** (уже в roadmap Фаза 5) — базовая ожидаемая функция
  для учётного приложения; для .xlsx у ассистента есть готовый навык.

**Средне по усилиям, высокая ценность:**
- **XIRR/TWR и линия «портфель vs бенчмарк (IMOEX/депозит)»** на графике динамики.
- **Max drawdown и волатильность** из снимков стоимости.
- **Налоговый блок для РФ**: оценка НДФЛ к уплате по закрытым сделкам, учёт ЛДВ
  (льгота на долгосрочное владение — по `openedAt/closedAt` видно срок 3+ года).
  Для российского частного инвестора это одна из самых прикладных фич.
- **Календарь дивидендов/купонов** (уже в roadmap) + прогнозная доходность.
- **Атрибуция валютного P&L**: сколько прибыли пришло от изменения цены бумаги, а
  сколько — от курса (данные `fxRate` по операциям уже есть).

**Позже / по запросу:**
- Целевое распределение и подсказки по ребалансировке (target allocation vs факт —
  круговая диаграмма распределения уже есть, не хватает «цели»).
- Ценовые алерты (в связке с WS-котировками из Фазы 4 roadmap).
- Выбор метода учёта FIFO ↔ средняя (сейчас движок только средневзвешенная).

## 4. Прочие изменения и улучшения

**Данные и корректность:**
- **Автоснимки стоимости по расписанию.** Сейчас снимок пишется только при ручном
  клике «Обновить цены» (по одному в день). Пропущенные дни → рваная equity-кривая и
  недостоверная просадка. Нужен фоновый снимок по расписанию (cron на бэке).
- **Сверка кэша (reconciliation).** Показывать расчётный денежный остаток и
  расхождение с фактическим у брокера — быстро ловит пропущенные/задвоенные операции
  до того, как они исказят P&L.

**UX-механика:**
- Инлайновое **редактирование операций** (не только удаление).
- **Сортировка + пагинация/виртуальный скролл** на всех таблицах; поиск по журналу.
- **Подтверждения** для необратимых действий (удаление, откат импорта).
- **Скелетоны** вместо «Загрузка…»; человеческие тексты ошибок.

**Внешний вид и платформа:**
- Вынести цвета в **токены темы**, добавить тёмную тему (заявлена, но не реализована).
- Проверить и починить **мобильную адаптивность** ключевых экранов.
- Заменить `calc(100vh - 64px)` на токен высоты тулбара (частично уже
  рефакторилось — см. недавние коммиты про flexbox).

**Порядок внедрения (рекомендация):**
1. Доходность в % и статистика сделок по системам (максимум смысла, данные готовы).
2. Глобальный фильтр система/портфель/период на дашборде.
3. XIRR + бенчмарк + max drawdown на графике динамики.
4. Редактирование операций + сортировка/поиск в таблицах.
5. Налоговый блок и экспорт.
