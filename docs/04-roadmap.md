# 04 — Дорожная карта

Порядок работ: от локального MVP к публичному SaaS. Каждая фаза — самостоятельный,
работающий результат. Не делаем всё сразу.

---

## Фаза 0 — Подготовка (0.5–1 день)

Цель: каркас проекта, чтобы было куда писать код.

- [ ] Инициализировать монорепо **Nx** (pnpm): `apps/web` (Angular 22) и `apps/api`
      (NestJS) + общая либа `libs/core` (типы, Zod-схемы, движок расчётов).
- [ ] Настроить строгий TypeScript, ESLint (angular-eslint) + Prettier, Vitest.
- [ ] Поднять пустой NestJS-сервер (Fastify-адаптер) и пустой Angular-фронт
      (`ng serve`), связать по REST API.
- [ ] Настроить Drizzle + SQLite, создать первую миграцию (пустая схема).

**Итог фазы:** `pnpm dev` поднимает фронт+бэк, открывается пустая страница.

---

## Фаза 1 — Ядро данных и расчётов (2–4 дня)

Цель: работает журнал операций и автоматический пересчёт — сердце системы.

- [ ] Описать схему БД (operations, systems, portfolios, instruments, trades,
      positions, quotes) из `02-data-model.md`.
- [ ] Реализовать **движок расчётов** в `libs/core`: средняя цена, вложено,
      реализованный/нереализованный P&L, статус сделки, агрегаты.
- [ ] **Покрыть движок тестами на ваших реальных цифрах** из примеров (SBER, TMON,
      LKOH, облигации). Это защита от ошибок в формулах.
- [ ] API-эндпоинты: CRUD операций, получение позиций и сделок (вычисляемых).
- [ ] Обработка переводов между портфелями (transfer_group).

**Итог фазы:** можно через API добавить операции и получить корректные позиции/P&L.

---

## Фаза 2 — Интерфейс MVP (3–5 дней)

Цель: то, чем вы реально пользуетесь вместо Google Sheets.

- [ ] Страница **Операции**: таблица (Angular Material Table / AG Grid) + адаптивная
      форма ввода на **Signal Forms** под тип операции (`03-ux-plan.md`, шаг 1).
- [ ] Страница **Портфель**: позиции с группировкой, итоговые плашки, фильтр по
      системе/портфелю, круговая диаграмма (шаг 3).
- [ ] Страница **Сделки**: открытые/закрытые, раскрытие в операции (шаг 2).
- [ ] Справочники: системы, портфели, инструменты (автокомплит тикеров).
- [ ] Форматирование чисел/валют/дат, цвет P&L.

**Итог фазы:** полноценная замена трёх листов Google Sheets, ввод в одном месте.

---

## Фаза 3 — Импорт брокеров (2–4 дня)

Цель: ваш сценарий «файл из брокера → программа сама разбирает».

- [ ] Общий интерфейс парсера + первый парсер (Tinkoff / T-Bank, .xlsx).
- [ ] Нормализация в единый формат + Zod-валидация.
- [ ] Движок классификации операций (покупка / перевод между портфелями /
      дивиденд / купон / налог / комиссия / депозит-вывод).
- [ ] Дедупликация и `import_batch_id` с возможностью отката.
- [ ] Страница **Импорт**: загрузка, предпросмотр, правка сомнительных строк
      (`03-ux-plan.md`, шаг 5).

**Итог фазы:** новые сделки заводятся загрузкой файла, а не ручным вводом.

---

## Фаза 4 — Рыночные данные по API (1–3 дня)

Цель: текущие цены и курсы подтягиваются сами (как вы выбрали).

- [ ] Коннектор **MOEX ISS API** — цены акций/облигаций/ETF.
- [ ] Коннектор **ЦБ РФ** — курсы валют.
- [ ] Коннектор крипты (Binance/CoinGecko) — BTC/USDT.
- [ ] Кэш котировок (`quotes`) + кнопка «Обновить цены» + fallback на ручную цену.
- [ ] Пересчёт текущей стоимости портфеля по свежим ценам.

**Итог фазы:** портфель показывает актуальную стоимость без ручного ввода цен.

---

## Фаза 5 — Дашборд и аналитика (2–3 дня)

Цель: обзорный экран и статистика (ваши будущие листы «Статистика»).

- [ ] **Дашборд**: сводные цифры, P&L по системам, топ прибыль/убыток
      (`03-ux-plan.md`, шаг 4).
- [ ] Снимки стоимости портфеля во времени → график динамики.
- [ ] Календарь купонов/дивидендов.
- [ ] Экспорт таблиц в Excel/CSV.

**Итог фазы:** аналитика поверх данных, а не отдельные ручные листы.

---

## Фаза 6 — Десктоп (1–2 дня)

Цель: приложение с иконкой, работает офлайн.

- [ ] Обернуть фронт в **Tauri 2**, БД — в папке приложения.
- [ ] Сборка под ваш ОС, автозапуск бэкенда внутри приложения.

**Итог фазы:** тот же продукт как обычная десктоп-программа. Код не переписывался.

---

## Фаза 7 — Публичный SaaS (отдельный крупный этап)

Делается только когда MVP устоялся и есть желание открыть доступ другим.

- [ ] Переключение БД SQLite → **PostgreSQL** (Drizzle упрощает миграцию).
- [ ] Мультипользовательность: аккаунты, изоляция данных по пользователю.
- [ ] Авторизация (Better Auth / аналог), роли.
- [ ] Хостинг бэка + БД, деплой фронта.
- [ ] Тарифы/лимиты, если монетизация.
- [ ] Безопасность: шифрование, аудит, бэкапы.

**Итог фазы:** тот же продукт доступен другим пользователям в интернете.

---

## Что делаем первым (ответ на «с чего начать»)

**Фаза 0 → Фаза 1 → Фаза 2.** То есть: каркас → движок расчётов с тестами на ваших
цифрах → три основных экрана. Уже после Фазы 2 вы полностью уходите из Google Sheets
для ручного учёта. Импорт (Фаза 3) и автоцены (Фаза 4) — сразу следом, они закрывают
вашу главную боль (ручной ввод).

Оценки в днях — ориентир для одного человека с ассистентом-генератором кода;
реальный темп зависит от того, сколько времени вы уделяете проекту.
