# 04 — Дорожная карта

Порядок работ: от локального MVP к публичному SaaS. Каждая фаза — самостоятельный,
работающий результат. Не делаем всё сразу.

---

## Фаза 0 — Подготовка (0.5–1 день)

Цель: каркас проекта, чтобы было куда писать код.

- [ ] Инициализировать монорепо **Nx** (pnpm): `apps/web` (Angular 22) и `apps/api`
      (NestJS) + общая либа `libs/core` (типы, Zod-схемы, движок расчётов).
- [ ] Настроить строгий TypeScript, ESLint (angular-eslint) + Prettier, Vitest.
- [ ] Поднять пустой NestJS-сервер (Fastify-адаптер) и пустой Angular-фронт
      (`ng serve`), связать по REST API.
- [ ] Настроить Drizzle + SQLite, создать первую миграцию (пустая схема).

**Итог фазы:** `pnpm dev` поднимает фронт+бэк, открывается пустая страница.

---

## Фаза 1 — Ядро данных и расчётов (2–4 дня)

Цель: работает журнал операций и автоматический пересчёт — сердце системы.

- [x] Описать схему БД (operations, systems, portfolios, instruments) из
      `02-data-model.md` — Drizzle + SQLite, `apps/api/src/db/schema.ts`.
- [x] Реализовать **движок расчётов** в `libs/core`: средняя цена, вложено,
      реализованный/нереализованный P&L, дивиденды/купоны.
- [x] **Покрыть движок тестами на ваших реальных цифрах** (SBER, TMON, LKOH) —
      `libs/core/src/lib/engine.spec.ts`, 6/6 проходят.
- [x] API-эндпоинты: CRUD операций (`OperationsService` на Drizzle), получение
      позиций; справочники систем/портфелей/инструментов (`PortfoliosModule`).
- [x] Обработка переводов между портфелями (transfer_group) — Transfer не
      искажает P&L, покрыто тестом.
- [ ] Таблицы `trades`, `positions` (снапшот), `quotes` — добавляются по мере
      необходимости в Фазе 2/4 (сейчас позиции считаются on-the-fly движком).

**Итог фазы:** через API можно добавить операцию и получить корректные позиции/P&L,
хранение — в реальной SQLite-базе (не in-memory).

**Как запустить локально:**
```bash
pnpm install
pnpm db:generate   # сгенерировать SQL-миграцию из schema.ts
pnpm db:migrate     # применить к data/portfolio.sqlite
pnpm db:seed        # (опционально) справочники: системы/портфели/инструменты
pnpm start:api      # http://localhost:3000/api
```

## Фаза 2 — Интерфейс MVP (3–5 дней)

Цель: то, чем вы реально пользуетесь вместо Google Sheets.

- [x] Страница **Операции**: таблица из реального API (httpResource) + адаптивная
      форма ввода на **Signal Forms** под тип операции (`03-ux-plan.md`, шаг 1).
- [x] Страница **Портфель**: позиции из API (движок @core на бэке), итоговые плашки.
      Группировка/фильтры/круговая диаграмма — доработка в этой же фазе.
- [x] API-клиенты на **httpResource + Zod** (FSD entities), dev-proxy /api→:3000.
- [x] Справочники: системы, портфели, инструменты (выпадающие списки в форме).
- [x] Форматирование чисел/валют (`libs/web-shared`).
- [ ] Страница **Сделки**: открытые/закрытые, раскрытие в операции (шаг 2).
- [ ] Портфель: группировка по типам, фильтр по системе/портфелю, круговая диаграмма.

**Итог фазы:** ввод операции в форме → сохранение в SQLite → пересчёт позиций
движком → отображение в портфеле. Полноценная замена ручного ввода в Google Sheets.

## Фаза 3 — Импорт брокеров (2–4 дня)

Цель: ваш сценарий «файл из брокера → программа сама разбирает».

- [x] Общий интерфейс парсера + первый парсер (универсальный CSV, `csv-parser.ts`).
- [x] Нормализация в единый формат (`libs/core/import.ts`), даты MM/DD/YYYY и
      DD.MM.YYYY, разделители разрядов.
- [x] Движок классификации операций (покупка/продажа/перевод между портфелями/
      дивиденд/купон/налог/комиссия/депозит-вывод) — с тестами на реальных данных.
- [x] Дедупликация (против БД и внутри файла) и `import_batch_id` с откатом.
- [x] Страница **Импорт**: загрузка, предпросмотр с подсветкой (OK/Проверить/Дубль/
      Ошибка), импорт, откат загрузки (`03-ux-plan.md`, шаг 5).
- [ ] Парсер родного .xlsx-отчёта Tinkoff (когда дадите образец файла).

**Итог фазы:** новые сделки заводятся загрузкой CSV, а не ручным вводом. Движок сам
определяет тип операции и переводы между портфелями, отсекает дубли.

> Пример файла для теста: `docs/example-import.csv`. Перед импортом справочники
> (системы/портфели/инструменты) должны существовать — см. `pnpm db:seed`.

## Фаза 4 — Рыночные данные по API (1–3 дня)

Цель: текущие цены и курсы подтягиваются сами (как вы выбрали).

- [x] Коннектор **MOEX ISS API** — цены акций/облигаций/ETF (`MoexProvider`, Adapter).
- [x] Коннектор **ЦБ РФ** — курсы валют (`CbrProvider`, кэш на 1 час).
- [x] Кэш котировок (таблица `quotes`) + кнопка «Обновить цены» на странице Портфель.
- [x] Пересчёт текущей стоимости и P&L в RUB по свежим ценам (`OperationsService`).
- [ ] Коннектор крипты (Binance/CoinGecko) — BTC/USDT (пока marketSource=manual).

**Итог фазы:** портфель показывает актуальную стоимость и P&L без ручного ввода цен.
Кнопка «Обновить цены» тянет котировки с MOEX и пересчитывает портфель.

> Перед первым запуском Фазы 4: `pnpm db:generate && pnpm db:migrate`
> (добавилась таблица `quotes`).

## Фаза 5 — Дашборд и аналитика (2–3 дня)

Цель: обзорный экран и статистика (ваши будущие листы «Статистика»).

- [x] **Дашборд** на Chart.js (ng2-charts): сводные плашки, поток/доход по месяцам

      Логика построения графиков — по `portfolio_dashboard.html` (палитра, цвет по

- [ ] Снимки стоимости портфеля во времени → линия динамики (нужна история —
      периодические снимки; пока timeline показывает поток/доход по месяцам).
- [ ] Календарь купонов/дивидендов.
- [ ] Экспорт таблиц в Excel/CSV.

**Итог фазы:** аналитика с графиками поверх данных, а не отдельные ручные листы.

> Требуются зависимости `chart.js` + `ng2-charts` — выполните `pnpm install`.

## Фаза 7 — Публичный SaaS (отдельный крупный этап)

Делается только когда MVP устоялся и есть желание открыть доступ другим.

- [ ] Переключение БД SQLite → **PostgreSQL** (Drizzle упрощает миграцию).
- [ ] Мультипользовательность: аккаунты, изоляция данных по пользователю.
- [ ] Авторизация (Better Auth / аналог), роли.
- [ ] Хостинг бэка + БД, деплой фронта.
- [ ] Тарифы/лимиты, если монетизация.
- [ ] Безопасность: шифрование, аудит, бэкапы.

**Итог фазы:** тот же продукт доступен другим пользователям в интернете.

## Что делаем первым (ответ на «с чего начать»)

**Фаза 0 → Фаза 1 → Фаза 2.** То есть: каркас → движок расчётов с тестами на ваших
цифрах → три основных экрана. Уже после Фазы 2 вы полностью уходите из Google Sheets
для ручного учёта. Импорт (Фаза 3) и автоцены (Фаза 4) — сразу следом, они закрывают
вашу главную боль (ручной ввод).

Оценки в днях — ориентир для одного человека с ассистентом-генератором кода;
реальный темп зависит от того, сколько времени вы уделяете проекту.
